Dataset Overview

Cross view geo-localization task aims to localize a query ground image by matching it to a GPS-tagged aerial image. This is a challenging task as there can be huge visual differences between ground view images and aerial view images at the same GPS location. Previous works have proposed several datasets containing cross-view image pairs to explore this task. However, these datasets only contain ground and aerial image pairs, and neglecting additional information such as geometry and semantics can be helpful to this task. In this work, we propose a new dataset that not only contains cross view image pairs, but also contains depth images and pixel-level semantic information. This dataset, for the first time, allows one to explore cross view geo-localization tasks with the help of depth and semantic information.
In this dataset, we collect aerial view images, aerial view depth images, aerial view semantic images with the simulated cameras, and ground-view images for every location. The ground view images are collected using google street view static API. For each GPS location, we collect 1 panorama ground image and 12 different perspective ground images with a view direction interval of 30 degrees. The aerial images are rendered from 3D meshes provided by SUM dataset , which also provides semantic labels for meshes. With the help of PyTorch3D, we render aerial images with corresponding depth images and semantic label images.
